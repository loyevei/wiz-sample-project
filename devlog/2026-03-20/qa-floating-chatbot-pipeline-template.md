# 플로팅 챗봇 5요소 파이프라인 QA 템플릿

- **대상 빌드**: `main.e7fa63189005f115.js`
- **검수 범위**: 플로팅 챗봇 최종 답변 파이프라인 (`프롬프트 + 오케스트레이터 + 도구 + 메모리 + 스트리밍 UI`)

## 1. 진입 / 기본 상태
- [ ] 우하단 플로팅 버튼이 정상 노출된다.
- [ ] 컬렉션 선택 화면이 정상 노출된다.
- [ ] 컬렉션 선택 후 채팅창 헤더/입력창/추천 질문 UI가 정상 표시된다.
- [ ] 기존 14단계 사고과정 카드가 여전히 함께 표시된다.

## 2. 파이프라인 카드 공통
- [ ] `최종 답변 구성 요소` 섹션이 별도 카드로 노출된다.
- [ ] 프롬프트 / 오케스트레이터 / 도구 / 메모리 / 스트리밍 UI 5개 카드가 모두 보인다.
- [ ] 각 카드의 상태 배지(`대기/진행 중/완료/건너뜀/오류`)가 자연스럽게 갱신된다.
- [ ] 파이프라인 카드가 참고 문헌/도구 로그/최종 답변 카드와 시각적으로 구분된다.

## 3. 프롬프트 레이어
- [ ] 질문 직후 프롬프트 카드가 먼저 진행 중 상태로 바뀐다.
- [ ] 응답 언어 또는 선택 컬렉션 정보가 카드 배지/상세 정보에 반영된다.
- [ ] 파이프라인 종료 시 프롬프트 카드가 완료 상태로 정리된다.

## 4. 오케스트레이터 레이어
- [ ] 질문 성격에 맞는 분류(예: 주제 발굴, 공정 예측, 진단 분석)가 표시된다.
- [ ] 계획 목록이 있을 때 `오케스트레이션 계획` 영역에 자연스럽게 표시된다.
- [ ] 도구 실행 전후로 오케스트레이터 상태가 적절히 갱신된다.
- [ ] 페이지 이동형 질문에서는 후속 페이지/탭 맥락이 카드에 반영된다.

## 5. 도구 레이어
- [ ] 도구 미사용 질문에서는 `건너뜀` 상태가 자연스럽다.
- [ ] 도구 사용 질문에서는 실행 개수/도구 이름 배지가 갱신된다.
- [ ] 별도 `도구 실행 로그` 섹션과 파이프라인 `도구` 카드가 서로 충돌하지 않는다.
- [ ] 검색 또는 이동 도구 실행 후 최종 답변 생성 흐름이 끊기지 않는다.

## 6. 메모리 레이어
- [ ] 첫 질문에서는 `새 대화` 또는 유사한 초기 상태가 자연스럽게 보인다.
- [ ] 두 번째 질문부터 이전 대화 턴 수 또는 최근 맥락 정보가 반영된다.
- [ ] 선택 컬렉션 정보가 메모리 카드에 유지된다.
- [ ] 검색 근거가 생긴 경우 메모리 카드 설명이 과도하게 깨지지 않는다.

## 7. 스트리밍 UI 레이어
- [ ] 답변 생성 중 스트리밍 UI 카드가 진행 중 상태를 표시한다.
- [ ] 텍스트가 점진적으로 출력될 때 레이아웃이 흔들리지 않는다.
- [ ] 완료 시 스트리밍 UI 카드가 완료 상태로 마무리된다.
- [ ] 긴 응답에서도 스크롤과 자동 하단 이동이 자연스럽다.

## 8. 통합 시나리오
### 시나리오 A: 참고 문헌 검색형
- 질문: 플라즈마 에칭 관련 최신 논문 핵심 포인트 정리해줘
- 기대 결과:
  - [ ] 참고 문헌 카드가 생성된다.
  - [ ] `도구` 카드가 실행 상태를 거쳐 완료된다.
  - [ ] `메모리` 카드에 검색 근거 반영 성격이 드러난다.
  - [ ] 최종 답변 카드가 별도 유지된다.

### 시나리오 B: 페이지 이동형
- 질문: OES 이상 탐지 흐름으로 이동해서 다음 액션까지 정리해줘
- 기대 결과:
  - [ ] 오케스트레이터 카드에 분류/페이지 맥락이 반영된다.
  - [ ] 도구 로그에 navigation 흔적이 남는다.
  - [ ] handoff 성격의 최종 답변이 함께 노출된다.

### 시나리오 C: 연속 대화형
- 질문 1: ICP 식각 조건 예측해줘
- 질문 2: 방금 조건 기준으로 RF 파워 영향만 더 설명해줘
- 기대 결과:
  - [ ] 두 번째 질문에서 메모리 카드가 이전 대화 맥락을 반영한다.
  - [ ] 이전 assistant 턴은 접히고 최신 턴만 펼쳐진다.
  - [ ] 파이프라인 카드 상태 갱신이 두 턴 모두 안정적이다.
