# 플로팅 챗봇 QA 템플릿

- **검수일**: 2026-03-20
- **검수자**: 
- **대상 빌드**: `main.9395a732229317b6.js`
- **검수 범위**: 플로팅 챗봇 14단계 사고과정 UI, 플라즈마 로봇 캐릭터, tool 카드, 최종 답변, 이전 턴 접기

## 1. 진입/기본 UI
- [ ] 우하단 플로팅 버튼이 정상 노출된다.
- [ ] 버튼/헤더에 플라즈마 로봇 캐릭터와 `PX14` 표시가 보인다.
- [ ] 채팅창 열기/닫기가 정상 동작한다.
- [ ] 컬렉션 선택 화면이 정상 노출된다.

## 2. 대화 흐름
- [ ] 질문 입력 후 사용자 메시지와 assistant 메시지가 순서대로 표시된다.
- [ ] 최신 assistant 턴만 펼쳐지고 이전 assistant 턴은 접힌다.
- [ ] 접힌 이전 턴을 다시 펼칠 수 있다.
- [ ] 연속 질문 2~3회 후에도 레이아웃이 깨지지 않는다.

## 3. 사고과정(14단계)
- [ ] 상단에 14단계 사고과정 아코디언이 표시된다.
- [ ] 현재 진행 단계가 시각적으로 구분된다.
- [ ] 단계 상태가 `pending/running/completed/error` 흐름으로 자연스럽게 갱신된다.
- [ ] 완료 시 현재 단계 설명이 최종 상태로 정리된다.

## 4. 중단 정보 패널
- [ ] 현재 단계 설명이 표시된다.
- [ ] 참고 문헌 목록이 있을 때 정상 표시된다.
- [ ] 참고 문헌이 없을 때 빈 상태 UI가 어색하지 않다.
- [ ] 유사도 요약이 표시된다.
- [ ] 긴 제목/긴 근거 텍스트에서도 카드 높이와 줄바꿈이 자연스럽다.

## 5. 도구 실행 로그
- [ ] `tool_use` 카드가 별도 영역에 표시된다.
- [ ] `tool_result` 카드가 별도 영역에 표시된다.
- [ ] 도구 로그가 최종 답변과 혼합되지 않는다.
- [ ] 여러 도구가 실행되어도 카드 순서가 자연스럽다.

## 6. 최종 답변
- [ ] 하단 최종 답변 영역이 별도 카드로 렌더링된다.
- [ ] 최종 답변이 참고 문헌/도구 카드와 시각적으로 구분된다.
- [ ] 긴 답변에서도 스크롤이 자연스럽다.

## 7. 네비게이션/연동
- [ ] 페이지 이동형 질문에서 handoff 응답과 이동 흐름이 자연스럽다.
- [ ] 컬렉션 선택 상태가 질문 후에도 유지된다.
- [ ] 에러 발생 시 에러 상태가 사용자에게 이해 가능하게 노출된다.

## 8. 추천 시나리오
### 시나리오 A: 참고 문헌 기반 질의
- 질문: 최근 플라즈마 진단 논문 기준으로 핵심 이슈 정리해줘
- 기대 결과:
  - [ ] 사고과정 단계가 진행된다.
  - [ ] 참고 문헌 목록이 생성된다.
  - [ ] 유사도 요약이 표시된다.
  - [ ] 최종 답변이 별도 카드로 정리된다.

### 시나리오 B: 페이지 이동형 질의
- 질문: 실험 데이터셋 페이지로 이동해서 조건-결과 관점으로 정리해줘
- 기대 결과:
  - [ ] tool 카드에 이동 관련 로그가 보인다.
  - [ ] 페이지 이동 또는 handoff 안내가 자연스럽게 표시된다.
  - [ ] 최종 답변이 분리되어 유지된다.

### 시나리오 C: 보고서형 질의
- 질문: 이 주제를 보고서 초안 형태로 정리해줘
- 기대 결과:
  - [ ] 단계 설명이 보고서 작성 흐름에 맞게 갱신된다.
  - [ ] 최종 답변이 구조화된 텍스트로 표시된다.
  - [ ] UI 깨짐 없이 긴 응답을 수용한다.
