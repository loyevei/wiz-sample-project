- [x] 파이프라인 카드가 참고 문헌/도구 로그/최종 답변 카드와 시각적으로 구분된다.
- 메모: 정상 상태에서는 파이프라인 카드가 2열 그리드로 정리되고, 하단 참고 문헌/도구/최종 답변 블록과 시각적 충돌이 없다.

## 3. 프롬프트 레이어

- [x] 응답 언어 또는 선택 컬렉션 정보가 카드 배지/상세 정보에 반영된다.
- [x] 파이프라인 종료 시 프롬프트 카드가 완료 상태로 정리된다.

## 4. 오케스트레이터 레이어

- [x] 계획 목록이 있을 때 `오케스트레이션 계획` 영역에 자연스럽게 표시된다.
- [x] 도구 실행 전후로 오케스트레이터 상태가 적절히 갱신된다.
- [x] 페이지 이동형 질문에서는 후속 페이지/탭 맥락이 카드에 반영된다.
- 메모: 정상 상태에서는 오케스트레이터 카드가 단순 상태 표시를 넘어서 다음 단계 계획과 handoff 방향을 설명해준다.

## 5. 도구 레이어

- [x] 별도 `도구 실행 로그` 섹션과 파이프라인 `도구` 카드가 서로 충돌하지 않는다.
- [x] 검색 또는 이동 도구 실행 후 최종 답변 생성 흐름이 끊기지 않는다.

## 6. 메모리 레이어

- [x] 두 번째 질문부터 이전 대화 턴 수 또는 최근 맥락 정보가 반영된다.
- [x] 선택 컬렉션 정보가 메모리 카드에 유지된다.
- [x] 검색 근거가 생긴 경우 메모리 카드 설명이 과도하게 깨지지 않는다.

  - [x] `도구` 카드가 실행 상태를 거쳐 완료된다.

### 시나리오 B: 페이지 이동형

  - [x] handoff 성격의 최종 답변이 함께 노출된다.
- 기대 화면 메모: `진단 분석`, `diagnosis`, `detection` 계열 문맥이 보이고, 이동 버튼/안내가 답변과 분리되어 있으면 정상이다.

  - [x] 이전 assistant 턴은 접히고 최신 턴만 펼쳐진다.
  - [x] 파이프라인 카드 상태 갱신이 두 턴 모두 안정적이다.
