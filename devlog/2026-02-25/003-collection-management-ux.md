# 컬렉션 관리 기능 검증 및 UI/UX 보완

- **ID**: 003
- **날짜**: 2026-02-25
- **유형**: 기능 추가

## 작업 요약
Embedding 페이지의 Milvus 컬렉션 추가/삭제 기능을 API 레벨에서 검증하고, UI/UX를 보완했다. 생성/삭제 중 로딩 상태 표시, 빈 컬렉션 안내 메시지, 프론트엔드 사전 유효성 검사, 삭제 확인 다이얼로그 상세화 등을 적용했다.

## 변경 파일 목록

### view.ts (UI 로직 보완)
- `creatingCollection`, `deletingCollection` 상태 변수 추가
- `createCollection()`: 프론트엔드 이름 유효성 사전 검사 + 중복 사전 검사 + 로딩 상태
- `deleteCollection()`: 삭제 확인 다이얼로그에 문서/청크 수 표시 + 삭제 중 로딩 상태
- `loadCollections()`: 삭제된 컬렉션 선택 상태 자동 초기화

### view.pug (UI 보완)
- 컬렉션 select 하단: 컬렉션 0개일 때 경고 안내 메시지
- 생성 버튼: 로딩 스피너 + "생성 중..." 텍스트, Enter 키 지원
- 컬렉션 테이블: 0개일 때 빈 상태 안내 + "첫 번째 컬렉션 만들기" CTA 버튼
- 삭제 버튼: 아이콘 추가, 삭제 중 스피너 표시 + 행 투명도 처리
- 컬렉션 목록 헤더: 개수 배지 + 새로고침 버튼 추가
